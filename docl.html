Command line interface

docl [options] --dir &lt;output-directory> FILES

Generate the html-documentation for the files FILES in the output-directory.

Before generating the documentation the first time, you need to create the output-directory. For example, to create the directory doc open a terminal and execute:

$ mkdir doc

Then you can generate the documentation by executing:

$ cd path/to/lua/files

$ ./path/to/docl/docl --dir doc *.lua

Options

--dir <ARG>
specify output directory. default is doc.
--lift <ARG>
lift a specific file. eg. make its documentation into the index page. you probably want to do that with the main file.
--suffix <ARG>
the <title> tag is constructed as MODULETITLE - SUFFIX. recommended value is MYPOGRAMM Documentation. default is Documentation.
--highlight <ARG>
choose a syntax-highligher. default is lua.
  • lua
    produces already colored html output, but big files.
  • js
    produces smaller files, but syntax-highlighting is only visible if js is enabled.
  • none
    no syntax-highlighting
--source <ARG>
choose to show the source code in the page default is false.
  • false
    dont include source code
  • true
    include a button on every page to show the source code.
-v
show version number and exit.
-h
show this help and exit.

Note

Docl names the documentation of files relative to the working directory.

If all files to be documented are inside a specific directory, you should change your directory there, before invoking docl.

For example if you are at /home/james/ and you call

$ docl my-programs/my-app/src/lua/*.lua

then docl will name the output files my-programs.my-app.src.lua.*.lua.html (which is pretty dumb).

if you want them to be called only *.lua.html change the directory before invoking docl.

$ cd my-programs/my-app/src/lua
$ docl *.lua

Examples

./docl *.lua

document all .lua files in this directory

./docl *.lua --lift main.lua */*.lua

document all .lua files in this directory and all subdirectories (but not recursive!). make main.lua the index file / startpage.

./docl --dir html-doc $$((find -name "*.lua"))
recursively document all files ending in .lua and put output into html-doc/
